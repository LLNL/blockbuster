Prerequisites

DMX is not required for running Blockbuster, but Blockbuster is intended to be used on DMX displays.  If you have a display cluster, you should already have DMX up and running.

Blockbuster can either use Xlib or OpenGL to render movie frames.  OpenGL is the preferred renderer so it's recommended that you have a hardware-accelerated implementation of OpenGL installed.  There are some optimizations for NVIDIA hardware, but any OpenGL implementation should suffice.

To compile with DMX support you'll need the dmxext.h header and libdmx.a library installed in the standard X directories.

Blockbuster uses the GTK interface so you must have GTK+ 1.2 installed to build it.

If you install the LibGen package by Cognigraph Software, you can make mpeg movies from sm files.

Downloading and Compiling Blockbuster, Sidecar and smtools:

Blockbuster can be downloaded from the SourceForge download page.

We are not responsible if you muck any of this up and it overwrites some critical libraries or otherwise causes you injury. You do not need to be root to install this software, so the potential for harm is minimal, but if Bad Things happen, tough noogies! We warned you. Do not hate on us.


After you've download and unpacked the archive, do the following:

  1. cd blockbuster-2.5

  2. Examine/edit Makefile to set the DMX and DEBUG variables appropriately. Important: BLOCKBUSTER DOES NOT BUILD WITH DMX BY DEFAULT -- YOU MUST SET DMX IN THE MAKEFILE TO BUILD WITH DMX SUPPORT!

  3. Optionally set the environment variable INSTALL_DIR. If you do not set INSTALL_DIR, it will be set to ./$SYS_TYPE, or if that is not defined, it will use ./$(uname -n), which on Linux machines is "Linux". Directories $INSTALL_DIR, $INSTALL_DIR/bin, $INSTALL_DIR/man, $INSTALL_DIR/lib, and $INSTALL_DIR/include will be created if needed.

  4. type make

  5. When finished, the blockbuster, sidecar, and *sm* executables will be in the $INSTALL_DIR/bin directory, Man pages are installed in $SYS_TYPE/man. Some incidental libraries and headers are also installed in $INSTALL_DIR/lib and $INSTALL_DIR/include. This is a bug in the installer that will be fixed in the future, but it is harmless.
